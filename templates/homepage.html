Coding Patterns in Django

About this site

I created Django Patterns.com in response to a talk proposal rejection from DjangoCon.us. Although I was not privy to the exact discussion, reviewers and organizers felt a previous talk I had presented at PyCon in 2011 presented “non-standard and controversial practices were framed as the normal and accepted ways of doing things”. Obviously this “raised concerns of misleading less experienced developers”.

As a result, I want to provide a space where solutions to common Django problems are presented thoroughly and discussed openly.

In short, this site’s goals are:

  • Provide a place look up a pattern for doing something
  • Get a clear explanation of how, when, why to use it
  • Provide for feedback and open discussion of the pattern by the community
  • Allow anyone to contribute patterns

How to contribute

There are several ways to contribute:

  • Comment and discuss patterns on the site
  • Suggest improvements to the site
  • Submit new patterns
  • Add to or enhance existing patterns

Comment and discuss patterns on the site

Please contribute your ideas, pros and cons of patterns using the Disqus comments form on each page.

Here are some guidelines:

  • Be courteous.
  • Comment on the pattern or idea, not the person submitting.
  • Be helpful in your feedback

Suggest improvements to the site

In order to be a good community resource, I welcome any ideas to improve the site. The best way is to open an issue on the GitHub repository for the site.

The site’s code is open source, and licensed under the BSD license, so feel free to contribute code, or just use the code for your own site.

Submit new patterns

The Django Patterns content is actually a Sphinx project. Its code is on GitHub. Fork the repository and add your pattern. There is an example_pattern.rst file with a basic outline. Then submit a pull request.

Every complete submission will be accepted, even if I personally disagree with the pattern. I reserve the right to make exceptions to what is accepted in the future, and will do so publicly. As editor of the site, I may edit your submission for clarity, grammar, and such.

Add to or enhance existing patterns

Follow the same instructions for submitting a new pattern: fork the djangopatterns repository, make the changes, and submit a pull request.
